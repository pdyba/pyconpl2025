üéØ What is Prompt Injection?

Prompt Injection is a security risk where malicious or cleverly designed user inputs manipulate a Large Language Model (LLM) into revealing, ignoring, or overriding its system instructions.

Core Risks:

  • Leakage of hidden system prompts or sensitive data
  • Policy bypass (e.g., ignoring safety rules or compliance filters)
  • Jailbreaks that change the assistant's role or behavior
  • Confusion of trust boundaries between user-provided data and system instructions
  • Increased attack surface for downstream integrations (APIs, tools, automation)

‚ö†Ô∏è Why Prompt Injection is Dangerous

  • LLM-powered apps often mix system prompts (trusted) with user inputs (untrusted)
  • Attackers exploit this boundary to access hidden instructions or sensitive flows
  • Real-world impact: data leakage, compliance failures, reputational harm, and financial risk
  • Requires structured testing and layered defenses across prompt design, input/output handling, and monitoring

üîü Prompt Injection in OWASP Top 10

The OWASP Top 10 for LLM Applications (2023) highlights Prompt Injection as one of the most severe risks.

Critical Vulnerability
  • Category: "Prompt Injection" is ranked as a critical vulnerability
  • Why it matters: It directly threatens confidentiality, integrity, and availability
  • Examples: Data exfiltration, ignoring safety guardrails, encoded payloads (Base64, JSON injection)
  • Connection to other OWASP risks: Training Data Poisoning, Model Denial of Service

üî§ Tokenization vs Vectorization

1. Tokenization

What it is:

Splitting text into smaller units (tokens).

Purpose:

Prepares raw text for further processing.

How it works:

  • Tokens can be words ("Apollo is hiring")
  • Tokens can be subwords ("Ap", "ollo")
  • Tokens can be characters ("A", "p", "o"...)

Example:

Text: "Apollo.io is hiring"

Tokens: [Apollo, ., io, is, hiring]

Key idea: Breaking text down into manageable parts.
VS

2. Vectorization

What it is:

Converting tokens (or text) into numerical representations (vectors).

Purpose:

Makes text understandable to machine learning models (which only work with numbers).

How it works:

  • One-hot encoding: Each word ‚Üí binary vector
  • TF-IDF: Each word ‚Üí weighted importance
  • Embeddings: Each token/word ‚Üí dense vector in continuous space

Example (word embeddings):

"Apollo" ‚Üí [0.21, -0.87, 0.34, ...]

Similar words have vectors that are close together.

Key idea: Turning pieces into numbers that models can work with.

üéØ Key Difference (in one line):

Tokenization = breaking text into pieces.
Vectorization = turning those pieces into numbers that models can work with.

üõ°Ô∏è Defensive Measures

Layered defense requires:

Prompt Design

Separate roles and avoid mixing system/user inputs

Input Sanitization

Escape special characters, validate formats

Output Filtering

Prevent disclosure of sensitive instructions

Immutable Instructions

Embed key rules in code, not in prompts

Red-teaming

Regular adversarial testing to surface new attacks
